行内标记

(细节)

行内标记允许文本内的单词和短语具有字符风格(如似斜体和粗体)和功能(如超链接)。

纯文本 典型结果 说明
*强调* 强调 通常渲染为斜体
**重点强调** 重点强调 通常渲染为粗体
`解释文本` (见右边的说明) 解释文本的渲染和含义取决于域或应用程序。它可以用于索引条目或显式描述标记(如程序识别符)。
``行内文本`` 行内文本 通常渲染为等宽文本。空格会被保留，但折行不会。
引用_ 引用 一个简单的一个单词的超链接引用。见超链接目标.
`短语引用`_ 短语引用 一个因含有空格或标点符号而需要被引号引用的超链接引用。见超链接目标.
匿名引用__ 匿名引用 使用两个下划线而非一个，简单和短语引用都可以是匿名的(引用文本的目标不能重复)。见 超链接目标.
_`行内内部目标` 行内内部目标 一个文本内的交叉引用目标。见 超链接目标.
|替代引用| (见右边的说明) 结果被替代定义所替代。它可以是一个文本、图片、超链接或这些的组合。
脚注引用 [1]_ 脚注引用 1 见 脚注.
引文引用 [CIT2002]_ 引文引用 [CIT2002] 见 引文.
http://docutils.sf.net/ http://docutils.sf.net/ 一个独立超链接。

星号、反引号、竖线和下划线为行内分隔字符。星号、反引号、竖线功能类似引号，成对围绕被标记单词或短语、空白或其他必须在它们外面的引用。如果你想将行内分隔字符作为文本使用，使用反斜杠转义或引用(使用两个反引号，如使用行内文本)它们即可。

具体来说，reStructuredText规范要求行内标记的开始和结束字符(行内标记分隔符)必须遵守以下规则:

  1. 开始字符必须开启一个文本块或紧跟在空格或  ' " ( [ {或 <中的任何一个之后。
  2. 开始字符之后必须紧跟非空白。
  3. 结束字符之前必须为非空白。
  4. 结束字符必须结束一个文本块(结束文档或后面跟一个空行)或其后紧跟空白或 ' " . , : ; ! ? - ) ] } / \ 或 >中的任何一个。
  5. 如果一个开始字符紧跟在  ' " ( [ {或 <其中的一个之后，它后面不能立即跟对应的字符  ' " ) ] }或 >。
  6. 开始字符和结束字符之间必须至少有一个字符。
  7. 一个未转义的反斜杠之后紧跟一个开始字符或结束字符会禁用标记识别，除非是行内文本的结束标记。

同时记住，行内标记不能嵌套(行内文本包含任何其他行内标记分隔字符除外，但这不算，因为没什么被处理了)。

使用反斜杠转义

(细节)

reStructuredText使用反斜杠("\")来覆盖指定给标记字符的特殊含义并获取文本字符本身。想获得一个文本反斜杠，使用转义的反斜杠("\\")。例如:

原始reStructuredText 典型结果
*escape* ``with`` "\" escape with ""
\*escape* \``with`` "\\" *escape* ``with`` "\"

在Python字符串中，转义任何反斜杠字符都是合适的，以便它们实际到达reStructuredText。 实现这个的最简单的方式是使用原始字符串:

Python字符串 典型结果
r"""\*escape* \`with` "\\"""" *escape* `with` "\"
 """\\*escape* \\`with` "\\\\"""" *escape* `with` "\"
 """\*escape* \`with` "\\"""" escape with ""

过渡

(细节)

纯文本 典型结果

A transition marker is a horizontal line
of 4 or more repeated punctuation
characters.

------------

A transition should not begin or end a
section or document, nor should two
transitions be immediately adjacent.

过渡标记是由4个或更多重复的标点符号组成的标记。

过渡不应开始或结束一个章节或文档，两个过渡也不应该直接相邻。

过渡常见于小说和短篇小说，作为一个间隔分隔一行或多行，标记文本区分或主题、时间、观点或强调变化。

显式标记

显式标记块用于漂浮的结构(脚注)、没有直接的文件表示(超链接目标、注释)或需要特殊处理(指令)。它们都以两个点和一个空格开始。
